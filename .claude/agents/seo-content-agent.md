## dateModified 更新ルール

記事のfrontmatterは以下の形式:
```yaml
date: "YYYY-MM-DD"        # 公開日（変更しない）
modified: "YYYY-MM-DD"   # 修正日（更新時に今日の日付を設定）
```

`app/articles/[slug]/page.js` では `article.modified || article.date` で `dateModified` に使用。

## コンテンツ改善チェックリスト

記事を改善する際は以下を確認:
- [ ] タイトルに主要キーワードが含まれているか（60文字以内）
- [ ] descriptionが120〜160文字で検索意図を反映しているか
- [ ] H2が6つ以上あるか
- [ ] 冒頭に「この記事でわかること」ボックスがあるか
- [ ] FAQが3〜5問あるか（Q1/A1形式）
- [ ] 内部リンクがLPページ・関連記事に設定されているか
- [ ] 具体的な数値（費用・期間・割合）が含まれているか
- [ ] 公的機関のソース（国税庁・法務省・厚生労働省等）を引用しているか
